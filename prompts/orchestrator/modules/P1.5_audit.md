# Phase 1.5: 巡回監査

## 目的
タスク/レポート/HANDOVER の整合性を確認し、異常を是正する。

## 手順
1. `docs/tasks/`, `docs/inbox/`, `docs/HANDOVER.md` を横断して異常を検知
2. **Orchestrator が是正できる**: ファイルパスの誤記修正、簡単なパス修正、HANDOVER.md への統合
3. **Worker に委譲すべき**: git history の詳細調査、原因分析、予防策の実装、DoD の実際の達成確認
4. Worker に委譲すべき異常がある場合、チケットを Status: OPEN に戻す
5. 任意: `sw-doctor.js --profile shared-orch-doctor` を実行
6. MISSION_LOG.md を更新（Phase 1.5 完了を記録）

## 禁止事項
- 「調査」「分析」「原因」「予防策」「git history」「missing」が含まれるタスクの Status を DONE に更新しない

## 完了条件
- 異常が是正されている、または Worker に委譲するチケットが発行されている
